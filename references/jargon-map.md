# Jargon Map

Use this reference when converting ordinary Chinese into Alibaba-style or broader Chinese internet-company jargon. It is based on public company-culture language and widely circulated internet-company phrasing, not confidential internal material.

## Tone Control

- **Light**: Keep normal grammar. Add only a few terms such as "对齐", "闭环", "抓手".
- **Standard**: Use business nouns and collaboration verbs, but keep each sentence understandable.
- **Heavy**: Stack frameworks, abstract nouns, and flywheel language for parody or "黑话拉满".

## Alibaba-Flavored Anchors

Use sparingly, only when relevant:

- 使命/愿景: "让天下没有难做的生意", "活 102 年".
- 价值观: "客户第一, 员工第二, 股东第三", "因为信任, 所以简单", "唯一不变的是变化", "今天最好的表现是明天最低的要求", "此时此刻, 非我莫属", "认真生活, 快乐工作".
- Organization/product context: "小前台大中台", "中台能力", "业务中台", "数据中台", "淘系", "天猫", "1688", "菜鸟", "大促", "双11", "小二".
- Addressing people: "同学", "一号位", "owner", "接口人".

## Sentence Templates

### General Rewrite

- `围绕 [目标], 我们先从 [场景/链路] 切入, 以 [抓手] 推动 [结果] 闭环。`
- `当前核心卡点是 [问题], 需要先拉齐 [口径/边界], 再推进 [动作] 落地。`
- `这件事可以沉淀成 [机制/SOP/方法论], 后续在 [场景] 里规模化复用。`
- `从用户心智和业务结果看, [动作] 能够撬动 [指标/收益]。`

### DingTalk Message

- `我先把 [材料/结论] 透传出来, 大家看下口径是否一致。`
- `[事项] 我来 owner, 今天先拉齐依赖, [时间] 前给到阶段性产出。`
- `这个点我先和 [团队/同学] 碰一下, 把边界和风险前置。`

### Weekly Report

- `本周围绕 [目标] 推进 [动作], 已完成 [结果], 初步跑通 [链路/机制]。`
- `当前风险主要集中在 [风险], 已通过 [动作] 做前置收口。`
- `下周会以 [抓手] 为核心, 拉通 [协作方], 推动 [里程碑] 闭环。`

### OKR or Review

- `O: 围绕 [业务目标] 建立可持续增长/提效机制。`
- `KR: 在 [时间] 前跑通 [链路], 将 [指标] 提升至 [数值/方向]。`
- `策略: 通过 [抓手一] + [抓手二] 的组合拳, 沉淀可复用打法。`

## Transformation Heuristics

1. Add context: "围绕...", "基于...", "从...维度看...".
2. Nominalize actions: "做 X" -> "推进 X 的落地闭环".
3. Add dimension splitting: "从业务、用户、组织协同三个维度拆解".
4. Add lifecycle: "先试点跑通, 再复盘沉淀, 最后规模化复制".
5. Add accountability: "我来 owner", "明确一号位", "拉齐接口人".
6. Add risk handling: "风险前置", "依赖收口", "口径校准".
7. Add value: "用户价值", "业务增量", "组织效率", "长期机制".

## Anti-Patterns

- Do not make the text unreadable unless the user explicitly wants parody.
- Do not replace every verb with jargon. Keep key nouns concrete.
- Do not invent fake Alibaba-specific customs, confidential acronyms, org names, or internal processes.
- Do not use "客户第一" to excuse rude or exploitative behavior.
- Do not add exaggerated metrics or commitments the source text did not contain.

## More Examples

Plain: `我需要你今天确认一下这个需求。`

Standard: `这个需求今天需要先拉齐口径, 麻烦你帮忙确认边界和优先级, 我这边好继续推进闭环。`

Plain: `我们开个会讨论方案。`

Standard: `我们拉一个会把方案口径对齐, 顺便把关键依赖、风险点和后续 owner 一起收口。`

Plain: `这个项目进度有点慢。`

Standard: `当前项目推进节奏低于预期, 主要需要把依赖项和责任边界再拉齐, 避免后续交付风险扩大。`

Plain: `这个活动效果很好, 可以继续做。`

Standard: `这次活动已经验证了正向业务收益, 建议复盘沉淀核心打法, 后续在相似场景里规模化复制。`

Plain: `我不知道为什么用户不喜欢。`

Standard: `当前用户体感和预期存在 gap, 需要从用户心智、产品链路和履约体验三个维度做归因拆解。`
